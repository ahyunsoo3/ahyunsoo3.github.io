Edge Functions

9 Deno TypeScript edge functions deployed on Supabase. 5 handle billing webhooks and purchase verification. 4 are API helpers for AI and content processing. All functions share a common _shared/ library for JWT auth and error handling.

Function cards

Detailed breakdown of each edge function's purpose, inputs, and side effects.

B1 · Billing
verify-apple
Validates an App Store receipt or server notification by calling Apple's /verifyReceipt API (legacy) or App Store Server API (StoreKit 2). Inserts a confirmed subscription row and grants monthly credits.
POST JWT required INSERT subscriptions Idempotent via nonce
B2 · Billing
verify-google
Verifies a Google Play purchase token using the Google Play Developer API. Checks purchaseState === 0 (purchased) and acknowledgementState === 1. Auto-acknowledges if needed.
POST JWT required INSERT subscriptions Idempotent via nonce
B3 · Billing
verify-lemonsqueezy
Processes Lemon Squeezy webhooks (order_created, subscription_created). Verifies HMAC-SHA256 signature using the signing secret. No JWT required — authentication is the webhook signature.
POST Webhook sig INSERT subscriptions
B4 · Billing
grant-credits
Awards one-time credit pack credits to user_credits.balance. Called after verify-apple or verify-google confirms a one-time purchase (not a subscription). Uses billing nonce to prevent double-grants.
POST JWT required UPDATE user_credits Nonce-protected
B5 · Billing
restore-purchase
Cross-device subscription restore. Re-queries App Store / Google Play with the latest receipt/token, checks expiry > now(), and upserts the subscription row. Called when user reinstalls or signs in on a new device.
POST JWT required UPSERT subscriptions
A1 · AI
tts-gemini
Converts practice question prompts to natural-sounding audio via Gemini 2.5 Pro TTS. Returns base64-encoded MP3. Credit check and deduction handled server-side. Audio cached in Supabase Storage for 24h.
POST Gemini 2.5 Pro Credit: 2/call Returns: MP3 base64
A2 · AI
stt-deepgram
Transcribes user speech (WAV or WebM) via Deepgram Nova-3 with language detection and confidence scoring. Transcript returned with word-level timestamps for pronunciation analysis.
POST Deepgram Nova-3 Credit: 3/call Returns: transcript + timestamps
A3 · AI
evaluate-writing
Sends essay + task prompt to ChatGPT-4o with structured output mode. Returns JSON rubric scored on the iBT 0–30 scale: Task Achievement, Coherence & Cohesion, Lexical Resource, Grammatical Range & Accuracy.
POST GPT-4o JSON Credit: 5/call Returns: rubric JSON
A4 · AI
generate-study-plan
Takes user's score history, weak section analysis, target date, and available study hours. Sends to Gemini 2.5 Pro in JSON mode to produce a structured weekly schedule (daily tasks with estimated durations).
POST Gemini JSON mode Credit: 3/gen Returns: weekly plan JSON

Shared library reference

The _shared/ directory contains utilities imported by every edge function. This keeps authentication, CORS handling, and error formatting consistent across all 9 functions.

_shared/auth.ts — JWT verification
import { createClient } from '@supabase/supabase-js'

// Called at the top of every edge function handler
export async function verifyJwt(req: Request): Promise<{ userId: string }> {
  const auth = req.headers.get('Authorization')
  if (!auth?.startsWith('Bearer ')) throw new AuthError('Missing token')

  const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY, {
    global: { headers: { Authorization: auth } },
  })
  const { data: { user }, error } = await supabase.auth.getUser()
  if (error || !user) throw new AuthError('Invalid or expired token')

  return { userId: user.id }
}
_shared/errors.ts — Typed error responses
export class AuthError    extends EdgeError { status = 401 }
export class PaymentError extends EdgeError { status = 402 }
export class NotFound     extends EdgeError { status = 404 }
export class ServerError  extends EdgeError { status = 500 }

// In every function's catch block:
} catch (err) {
  if (err instanceof EdgeError) {
    return new Response(JSON.stringify({ error: err.message }), {
      status: err.status,
      headers: { 'Content-Type': 'application/json', ...corsHeaders },

auth.ts exports
  • verifyJwt(req) → { userId } or throws AuthError
  • getServiceClient() → service-role Supabase client for privileged writes
  • getUserCredits(userId) → current credit balance
  • deductCredits(userId, amount) → atomic deduction
cors.ts exports
  • corsHeaders — standard CORS headers object
  • handleOptions(req) — responds to OPTIONS preflight immediately
  • All functions call handleOptions before any auth check

Error handling

All edge functions return structured JSON errors with consistent HTTP status codes. The Flutter core_network layer maps these to typed NetworkException subclasses.

HTTPError classTriggerFlutter handling
400BadRequestErrorMissing required fields, invalid payloadShow validation message to user
401AuthErrorMissing / expired / invalid JWTSilent token refresh → retry; if refresh fails → sign-out
402PaymentErrorInsufficient AI creditsNavigate to billing screen with upgrade prompt
404NotFoundErrorResource not found (question, session)Show "not found" state UI
409ConflictErrorDuplicate nonce (replay attack attempt)Silently ignore — purchase already processed
422ProviderErrorAI provider returned invalid responseTrigger retry with backoff
500ServerErrorUnexpected exception in edge functionShow generic error with retry button
503RetryExhaustedAll 10 retry attempts failedShow "service unavailable" with try-later message
N
Credits are never deducted on failure Credit deduction (UPDATE user_credits) only occurs after the AI provider returns a successful response. If the provider fails after all retries, no credits are deducted and 503 is returned to the client.
