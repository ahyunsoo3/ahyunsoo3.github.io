core_network API

core_network wraps Dio with auth, retry, and error mapping. All feature modules use NetworkClient — never raw Dio.

NetworkClient usage (Dart)
// Provided via Riverpod
final networkClientProvider = Provider((ref) => NetworkClient(
  baseUrl: Env.supabaseUrl,
  interceptors: [
    AuthInterceptor(ref),   // attaches JWT, refreshes on 401
    RetryInterceptor(),     // exponential backoff for 5xx
    LogInterceptor(),       // dev-only request/response logging
  ],
));

// In a repository
class SpeakingRepository {
  final NetworkClient _client;

  Future<TranscriptResult> transcribeSpeech(Uint8List audio) async {
    final response = await _client.post(
      '/functions/v1/stt-deepgram',
      data: FormData.fromMap({ 'audio': MultipartFile.fromBytes(audio) }),
    );
    return TranscriptResult.fromJson(response.data);
  }
}
NetworkException hierarchy
NetworkException                   // base class
├── UnauthorizedException          // 401 → triggers token refresh
├── PaymentRequiredException       // 402 → triggers billing screen
├── NotFoundException              // 404 → show "not found" UI
├── ConflictException              // 409 → idempotent retry (ignore)
├── ProviderException              // 422 → AI provider error, retry
├── ServerException                // 500 → generic error message
├── ServiceUnavailableException    // 503 → "try again later" UI
└── TimeoutException               // timeout → show retry button

Usage examples

Typical usage patterns for each package. All three packages are consumed via Riverpod providers, keeping feature modules decoupled from package internals.

core_ui — using MRButton and ScoreRing in a widget
import 'package:core_ui/core_ui.dart';

class WritingFeedbackScreen extends StatelessWidget {
  final WritingResult result;

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        ScoreRing(
          score: result.overallScore,
          maxScore: 30,
          label: 'Overall',
          animate: true,
        ),
        MRCard(
          child: Text(result.feedback, style: AppTheme.bodyText),
        ),
        MRButton.primary(
          label: 'Practice Again',
          onPressed: () => context.go('/writing'),
        ),
      ],
    );
  }
}
core_billing — subscribing to a plan
import 'package:core_billing/core_billing.dart';

// In the billing module's notifier
class BillingNotifier extends AsyncNotifier<BillingState> {
  late final BillingService _service;

  @override
  Future<BillingState> build() async {
    _service = ref.watch(billingServiceProvider);
    final plan = await _service.getCurrentPlan();
    final products = await _service.getProducts();
    return BillingState(currentPlan: plan, products: products);
  }

  Future<void> subscribe(PlanTier tier) async {
    state = const AsyncValue.loading();
    final result = await _service.purchaseSubscription(tier);
    if (result.success) {
      state = AsyncValue.data(state.value!.copyWith(currentPlan: tier));
    }
  }
}

Testing shared packages Each package has its own test suite. core_ui uses Flutter widget tests with golden file comparison. core_network uses unit tests with a mock Dio adapter. core_billing uses a MockBillingService to test all purchase flows without hitting a real store — including simulating purchase failures, refunds, and restore flows.
