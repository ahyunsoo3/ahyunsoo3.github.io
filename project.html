Flutter app

14 feature modules, 1 coherent product

Each module has its own data layer, business logic, and UI — but shares auth, theme, routing, and billing through cross-cutting services and shared packages.

lib/features/auth/ Authentication

Social auth & session gate

SocialAuthService handles four sign-in paths: native Google on Android, Supabase OAuth Google on web/Android, native Apple on iOS/macOS, and email/password. Each path produces a Supabase session via signInWithIdToken or signInWithOAuth. AppSessionGate wraps the router and blocks rendering until the session is resolved, preventing any screen flicker on cold start.

  • Cross-platform OAuth with platform-specific fallback paths
  • Nonce-based Apple ID token validation against Supabase
  • Nickname onboarding flow after first sign-in
  • Auto-login service for seamless return sessions
lib/features/onboarding/ Onboarding

First-run flow & mobile web install

FirstRunOnboardingFlow shows a welcome screen and three randomly-picked Writing cloze tasks so users feel the product before signing in. After demo tasks, the SignInGatePage collects credentials. A separate MobileWebInstallPrompt detects if the user is on a mobile browser and conditionally suggests installing the PWA — only shown when it makes sense per platform.

  • Demo-before-signup reduces sign-in friction
  • Platform-aware PWA install prompt (iOS Safari / Android Chrome)
  • SharedPreferences tracks onboarding state across restarts
lib/features/reading/ Reading practice

Three reading task types

Academic passage: long passage with multiple questions, scroll-linked progress. Daily life: email and document formats with multiple-choice questions. Word completion: fill-in-the-blank sentences with adaptive difficulty. Each page fetches its content from Supabase and persists results to practice history.

  • Task type router: one entry point dispatches to correct page
  • Adaptive difficulty adjusts based on recent accuracy history
  • Correct/incorrect feedback written to Postgres on submit
lib/features/listening/ Listening practice

Four listening task types with remote audio

Conversation, academic talk, announcement, and choose-a-response pages. Audio is stored in a Supabase Storage bucket; the Flutter app resolves public URLs at runtime using the DB path. The audio player enforces listen-before-answer ordering for realistic exam simulation.

  • Supabase Storage path → public URL resolved at runtime, not hardcoded
  • Audio playback gating before MCQ answer submission
  • 5 MCQs per academic talk, 2 per conversation — matches real TOEFL format
lib/features/speaking/ Speaking practice

Speech recognition via Deepgram

TakeInterviewPage and ListenRepeatPage record user audio then send WAV bytes to DeepgramListenClient, which calls Deepgram's POST /v1/listen with Nova-3 (English monolingual). The transcript is then optionally passed to GeminiPracticeAssessmentClient for AI scoring. Token usage is logged to record_ai_token_usage for quota tracking.

  • WAV audio → Deepgram STT → transcript for scoring
  • AI credit consumed only if a transcript is successfully produced
  • API key resolved from .env asset, compile-time define, or dotenv
lib/features/writing/ Writing practice

Three writing task types with AI scoring

Email, academic discussion, and cloze (build a sentence) tasks. Email and academic discussion submissions are scored by GeminiPracticeAssessmentClient with a tier-aware model (Hyper tier gets gemini-3.1-pro-preview; others get gemini-3.1-flash-lite-preview). Results display a band-6 model answer alongside the AI score.

  • Input truncated to <1000 tokens to control API cost
  • Retry logic: up to 10 attempts with 4-second backoff
  • Model answer card shown after scored attempts
lib/features/assessment/ AI scoring engine

GeminiPracticeAssessmentClient

Singleton HTTP client that calls the Gemini generateContent API. Handles: API key resolution, input token budget (≈4 chars/token, truncated to 1000), output capped at 1000 tokens, retry loop up to 10 attempts, tier-based model selection, and async usage logging to Postgres for cost tracking. A ScoringRetryProgressCallback drives a live progress UI in the app.

  • Tier-aware model routing (Flash vs Pro preview)
  • Structured JSON output parsed into typed Dart assessment models
  • AI credit deducted before call; usage logged after
lib/features/statistics/ Progress analytics

Expected score estimation

The statistics page aggregates scored practice runs across all four sections and estimates an expected TOEFL score. It uses 1 AI credit to generate the estimate from a sample of the user's practice history. Coverage (percentage of 12 task types that have scored runs) and validity signal are shown alongside the score estimate.

  • Score validity rises with more scored runs per task type
  • Confirmation dialog before consuming the AI credit
  • Item coverage matrix: 12 task types × cleared/open state
lib/features/practice/ Adaptive difficulty

Adaptive study level engine

AdaptiveStudyLevel looks at the last 12 graded practice entries per section and computes accuracy. Below 60% → signal review → drop one level. Above 88% → signal stretch → raise one level. Between → signal steady → hold. AdaptiveStudyLevelConfidence tracks whether there is enough data (≥6 answers) or confident data (≥20 answers) before shifting the level.

  • Accuracy thresholds: <60% review · ≥88% stretch
  • Conservative shifts: moves at most one level per recommendation
  • Confidence guard: won't shift without ≥6 graded answers
lib/features/home/ · lib/features/subscription/ Monetization

Store page & billing clients

StorePage shows subscription plans and one-time credit packs. Five billing clients handle each platform: AppStoreSubscriptionBilling, GooglePlaySubscriptionBilling, LemonSqueezyBillingClient, AppStoreStorePackBilling, and GooglePlayStorePackBilling. All flows use a server-side nonce to prevent replay attacks on purchase tokens.

  • Platform detected at runtime: iOS, Android, or web → correct billing client
  • Nonce issued from server before purchase; consumed on verification
  • Plan limits enforce AI credit quotas per subscription tier
lib/features/version_gate/ Version control

Force-update guard

ForceUpdateGuard wraps the entire app router. On startup it queries AppVersionGateRepository which fetches the minimum required version from Supabase. If the installed version (read from the build-baked AppBuildVersion) is below the minimum, the app renders a full-screen block with a deep-link to the store. If the policy check fails, the app stays blocked — stale binaries cannot fail open.

  • Fail-closed: network error keeps app blocked, not open
  • Platform matrix: separate min versions for iOS, Android, web, macOS
  • Test hook: platformOverride + versionLabelLoader for unit tests
lib/features/history/ · lib/features/scoring/ History & scoring

Practice history & bookmarks

All practice attempts — correct counts, task types, mock vs practice mode, scoring results — are written to Postgres and surfaced in PracticeHistoryPage. PracticeBookmarksPage lets users save items for review. ScoringHistoryPage shows a timeline of AI-scored runs with their results.

  • Scoring pending / failed / success states tracked per entry
  • Bookmark toggle synced to Supabase in real time
  • Mock exam mode creates a separate batch entry for full-test tracking
lib/features/mock/ Mock exam

Full timed mock exam

MockExamPage sequences all four sections with real TOEFL timing. MockTransitionPage handles section breaks. Results are stored as a single grouped practice history batch, separable from individual practice sessions in the statistics view.

  • Section sequence: Reading → Listening → Speaking → Writing
  • Timed section enforcement with auto-advance on expiry
  • Separate result grouping: mock vs practice in history
lib/features/chat/ AI tutor

AI tutor chat

TutorChatAppBarButton surfaces an AI chat assistant accessible from any practice page. Credits are drawn from the tutor_chat_credits balance in the user profile. The tutor is aware of the current practice context, providing section-specific hints and explanations.

  • Tutor credit pack purchasable separately from scoring credits
  • App bar integration: available in-context from any practice flow

Shared packages

Three packages, four apps

Shared packages enforce a single source of truth across TOEFL, SAT, IELTS, and GRE. Any billing rule change, design token, or Supabase config fix propagates to all apps from one place.

packages/core_ui/

Design system

Material 3 light and dark themes (CoreTheme), typography scale (CoreTypography), and shared button components (CorePrimaryButton, CoreSecondaryButton). All visual consistency in the multi-app family flows from this package.

packages/core_network/

Supabase networking

SupabaseConfig and SupabaseClientFactory centralise URL and key resolution. Repository interfaces define contracts for Supabase-backed data access without coupling feature code to concrete implementations.

packages/core_billing/

Billing primitives

Free, Plus, Pro, and Hyper subscription tier scaffolding. Credit wallet logic (aiScoringCredits, tutorChatCredits, liveCaptionsSeconds). Repository and service contracts for app-specific billing persistence.

Supabase edge functions

9 Deno TypeScript functions

All deployed as Supabase Edge Functions. Each has its own index.ts and imports from a shared _shared/ library for billing logic, catalog lookups, and response helpers.

verify-app-store-subscription Billing

Verifies an Apple receipt against the App Store API using a nonce for replay protection. Maps the product_id to a canonical plan (plus/pro/hyper) via app_store_catalog.ts, then calls applyAppStoreSubscriptionFromReceiptRow to update user_profiles.

verify-google-play-subscription Billing

Authenticates with the Google Play Developer API using a service account JSON (google-auth-library), fetches subscriptionsv2 for the purchase token, and applies the subscription via applyGooglePlaySubscriptionFromV2.

lemon-squeezy-webhook Billing

Handles all Lemon Squeezy events: subscription created/updated/expired and order created. Verifies HMAC webhook signature, maps variant IDs to plan IDs, checks renewal dates for usage period reset, and grants one-time store packs with idempotent order tracking.

verify-app-store-store-pack Billing

Verifies one-time App Store purchases (AI scoring credits, tutor messages, live captions). Maps product ID to a StorePackGrant (e.g. 40 or 100 AI scoring credits) and calls applyStorePackGrantForProfile.

verify-google-play-store-pack Billing

Same pattern as App Store store pack but using Google Play's Products API (inappproducts) to verify one-time purchases and grant credits.

create-lemon-squeezy-checkout API

Creates a Lemon Squeezy checkout URL for web purchases. Embeds auth_provider, external_user_id, and plan_id in custom data so the webhook can identify the user without requiring them to be logged in on the payment page.

create-lemon-squeezy-customer-portal API

Generates a Lemon Squeezy customer portal URL for subscription management (upgrade, downgrade, cancel). Looks up the stored lemon_squeezy_customer_id from user_profiles and returns a one-time portal URL.

exchange-rates API

Fetches current currency exchange rates from an external provider and returns them for display in the pricing and store pages. Results may be cached at the edge to reduce upstream API calls.

exchange-convert API

Performs server-side currency conversion using stored exchange rates, returning a converted amount for a given source currency, target currency, and input value.

Billing system

Three providers, one user profile

Whether a user pays on iOS, Android, or web, the subscription state lands in the same user_profiles row. Each path is idempotent and replay-safe.

Replay-safe nonce system

Before any purchase, the client requests a short-lived nonce from the server. The nonce is stored in lemon_squeezy_billing_nonces with an expiry and a purpose field (e.g. google_play_plan). The purchase token is then sent to the Edge Function along with the nonce. The function validates the nonce, verifies the token with the store API, and deletes the nonce before updating the profile — preventing double-grants on webhook retries or replayed requests.

→
2 User pays in store
→
3 Token + nonce sent to Edge Function
→
4 Nonce validated & deleted
→
5 Store API verification
→
6 user_profiles updated

AI integration

AI scoring, speech recognition, content generation, and tap dictionary

G

Gemini AI scoring

GeminiPracticeAssessmentClient

Scores Writing (email, academic discussion) and Speaking tasks. Two model tiers: gemini-3.1-flash-lite-preview for Free/Plus/Pro; gemini-3.1-pro-preview for Hyper. Input is capped at ~1,000 tokens (4 chars/token estimate) to control cost. Output is capped at 1,000 tokens. Up to 10 retry attempts with 4-second backoff handle transient API errors. All calls are logged with input/output token counts for cost tracking.

Prompt engineering Token budget management Retry with backoff Cost tracking
D

Deepgram speech recognition

DeepgramListenClient

Sends WAV audio bytes to Deepgram POST /v1/listen with Nova-3 English model for pre-recorded STT. The client handles three API key resolution paths: bundled .env asset, compile-time --dart-define, and dotenv fallback. A BOM stripping step handles encoding edge cases in keys. Approximate STT token usage is logged to record_ai_token_usage.

Multi-path secret resolution WAV audio handling Usage logging
G

Content generation

scripts/content_ingestion/gemini_generate.py

Uses gemini-3.1-flash-lite-preview (overridable via GEMINI_MODEL env) to generate JSONL practice content for all 12 task types. Each task type has a schema derived from Pydantic models — Gemini is prompted to produce JSON Lines matching that schema. After generation, the ingestion pipeline validates and loads the rows.

Structured output prompting Schema-driven generation
C

ChatGPT tap dictionary

OpenAiTapDictionaryClient

Tapping a word in Reading sends the selection and nearby sentence context to ChatGPT through the same credit-checked path as other AI features. The model returns a short gloss—sense, register, and a minimal example—appropriate for TOEFL-level study. Responses can be cached per passage to reduce repeat latency and cost; usage is logged with input/output tokens like Gemini and Deepgram calls.

Context-aware definitions Credit-gated requests Usage logging

End-to-end flows

From tap to database — traced in full

Auth flow (Google, iOS native example)

1User taps "Sign in with Google"
2SocialAuthService.signInWithGoogle() — platform check: iOS → native Google Sign-In SDK
3Google returns GoogleSignInAccount with an ID token and raw nonce
4supabase.auth.signInWithIdToken() with hashed nonce — Supabase validates with Google
5Supabase creates / updates auth.users row; returns session
6AppSessionGate resolves; router navigates to onboarding or home
7If first sign-in: NicknameFlow prompts for display name → writes to user_profiles

AI scoring flow (Writing email)

1User finishes writing and taps "Submit for scoring"
2WritingEmailAssessor checks subscription plan via PlanLimits — gates if no quota
31 AI scoring credit deducted from user_profiles.ai_scoring_credits before calling API
4Input truncated to ≤1000 tokens; model selected based on subscription tier
5Gemini generateContent called — up to 10 retries with 4s backoff on failure
6JSON response parsed into WritingEmailAssessment model
7Score + model answer saved to practice history in Postgres
8Result card shown with score, feedback, and band-6 model answer

Subscription billing flow (Google Play)

1User taps "Subscribe — Pro" in StorePage
2GooglePlaySubscriptionBilling requests a nonce from lemon_squeezy_billing_nonces via Edge Function
3Google Play billing UI shown; user completes payment
4Purchase token + nonce sent to verify-google-play-subscription Edge Function
5Edge Function authenticates with Google Play API via service account, fetches subscriptionsv2
6Nonce validated and deleted — replay protection
7user_profiles updated: plan = "pro", usage period started, renewal date stored
8Flutter refreshes subscription state; Pro features unlock immediately
